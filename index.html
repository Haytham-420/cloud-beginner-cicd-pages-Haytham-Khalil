What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. Continuous Integration means that every time a developer pushes code to a repository, an automated pipeline runs to check, test, and validate the code immediately. Continuous Deployment means that once the code passes those checks, it is automatically deployed to a live environment without any manual steps. This process ensures that software is always in a deployable state and reduces human error.

My Pipeline

When I push code to the main branch on GitHub, a GitHub Actions workflow is triggered automatically. The workflow first checks that all required files (index.html, style.css, script.js) exist, then it uploads the site files and deploys them to GitHub Pages, making the updated website live within seconds.
